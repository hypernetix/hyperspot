# Streaming gRPC

## Upstream Configuration

```json
{
  "id": "gts.x.core.oagw.upstream.v1~user.service.grpc.v1",
  "server": {
    "endpoints": [
      { "scheme": "grpc", "host": "grpc.example.com", "port": 443 }
    ]
  },
  "protocol": "gts.x.core.oagw.protocol.v1~x.core.grpc.v1",
  "auth": {
    "type": "gts.x.core.oagw.plugin.auth.v1~x.core.oagw.apikey.v1",
    "config": {
      "header": "x-api-key",
      "secret_ref": "vault://secrets/user-service/api-key"
    }
  },
  "rate_limit": {
    "rate": 100,
    "window": "minute"
  }
}
```

## Route Configuration

```json
{
  "id": "gts.x.core.oagw.route.v1~user.service.list_users.v1",
  "upstream": "gts.x.core.oagw.upstream.v1~user.service.grpc.v1",
  "match": {
    "grpc": {
      "service": "user.v1.UserService",
      "method": "ListUsers"
    }
  },
  "plugins": [
    "gts.x.core.oagw.plugin.filter.v1~x.core.oagw.logging.v1"
  ]
}
```

## Inbound Request (HTTP/JSON - gRPC transcoding)

```http
POST /api/oagw/v1/proxy/gts.x.core.oagw.route.v1~user.service.list_users.v1 HTTP/1.1
Host: oagw.example.com
Authorization: Bearer <tenant-token>
Content-Type: application/json
Accept: application/x-ndjson
X-Request-ID: req-grpc-stream-001

{
  "page_size": 10,
  "filter": {
    "status": "ACTIVE"
  }
}
```

## Outbound Request (gRPC to upstream)

```
# gRPC HTTP/2 Request
:method: POST
:path: /user.v1.UserService/ListUsers
:authority: grpc.example.com
:scheme: https
content-type: application/grpc
x-api-key: <secret-from-vault>
x-request-id: req-grpc-stream-001
te: trailers
grpc-accept-encoding: gzip

# Protobuf-encoded message (ListUsersRequest)
[Binary: page_size=10, filter.status=ACTIVE]
```

## Upstream Response (gRPC server streaming)

```
# gRPC HTTP/2 Response Headers
:status: 200
content-type: application/grpc
grpc-encoding: identity

# Stream Message 1 (User)
[Binary: id="usr_001", name="Alice Smith", email="alice@example.com", status=ACTIVE]

# Stream Message 2 (User)
[Binary: id="usr_002", name="Bob Jones", email="bob@example.com", status=ACTIVE]

# Stream Message 3 (User)
[Binary: id="usr_003", name="Carol White", email="carol@example.com", status=ACTIVE]

# gRPC Trailers
grpc-status: 0
grpc-message: OK
```

## Bidirectional Streaming Example

**Route Configuration** (bidirectional):

```json
{
  "id": "gts.x.core.oagw.route.v1~chat.service.stream.v1",
  "upstream": "gts.x.core.oagw.upstream.v1~chat.service.grpc.v1",
  "match": {
    "grpc": {
      "service": "chat.v1.ChatService",
      "method": "Chat"
    }
  }
}
```

**Client sends** (newline-delimited JSON):

```http
POST /api/oagw/v1/proxy/gts.x.core.oagw.route.v1~chat.service.stream.v1 HTTP/1.1
Host: oagw.example.com
Authorization: Bearer <tenant-token>
Content-Type: application/x-ndjson
Accept: application/x-ndjson
X-Request-ID: req-bidi-001
Transfer-Encoding: chunked

{"message":"Hello"}
{"message":"How are you?"}
{"message":"Tell me a joke"}
```

**Server responds** (newline-delimited JSON):

```http
HTTP/1.1 200 OK
Content-Type: application/x-ndjson
X-Request-ID: req-bidi-001
Transfer-Encoding: chunked

{"reply":"Hi there!","timestamp":"2026-01-26T18:52:01Z"}
{"reply":"I'm doing great, thanks!","timestamp":"2026-01-26T18:52:02Z"}
{"reply":"Why did the programmer quit? Because he didn't get arrays!","timestamp":"2026-01-26T18:52:03Z"}
```

## Notes

- OAGW performs gRPC-JSON transcoding for HTTP clients
- Protobuf ↔ JSON conversion based on proto descriptors (configured per upstream)
- Server streaming returns `application/x-ndjson` (newline-delimited JSON)
- Bidirectional streaming uses chunked transfer encoding in both directions
- gRPC metadata maps to HTTP headers with `grpc-metadata-` prefix
- gRPC status codes map to HTTP status codes (0 → 200, etc.)
- For native gRPC clients, OAGW can act as a pure gRPC proxy without transcoding
